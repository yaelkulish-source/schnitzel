## סקירה כללית

מערכת ניהול הזמנות לדוכן שניצלים בקיבוץ.
פועל כל יום שלישי 18:00–20:30.
שני טאבלטים מסונכרנים בזמן אמת + טופס הזמנה ללקוחות דרך קישור בווצאפ.

## ארכיטקטורה

### מסכים
| מסך | מכשיר | תפקיד |
|-----|--------|--------|
| **קבלה** | טאבלט 1 | הכנסת הזמנות ידנית, ניהול סטטוסים, תשלום |
| **ציפסרים** | טאבלט 2 | תצוגה בלבד — מה להכין ובאיזה סדר |
| **טופס לקוח** | כל טלפון | הלקוח פותח קישור מווצאפ, ממלא טופס בדפדפן |

### ערוץ הזמנות ווצאפ
- את שולחת קישור לטופס בווצאפ
- הלקוח פותח בדפדפן, ממלא תוך 30 שניות
- ההזמנה נכנסת אוטומטית למסך הקבלה
- **אין חיבור ישיר לווצאפ בשלב 1**

### טכנולוגיה
- **Backend:** Node.js + WebSocket לסנכרון real-time בין הטאבלטים
- **Frontend:** HTML/CSS/JS — ללא framework מורכב
- **Database:** שומר כל הזמנה עם תאריך (לתמיכה בהיסטוריה)
- **Hosting:** GitHub Pages (frontend) + Railway/Render (backend) — בחינם
- **שפה:** עברית בלבד, RTL מלא
- **אין MCP בשלב 1** — הסנכרון נעשה דרך WebSocket בקוד רגיל

### סנכרון ואמינות
- נתונים משותפים בזמן אמת בין כל המסכים דרך WebSocket
- **Offline first** — שומר הזמנות מקומית (IndexedDB), מסנכרן כשחיבור חוזר
- **גיבוי אוטומטי** — כל שינוי נשמר מיד לשרת
- **גיבוי ידני** — כפתור להורדת קובץ של כל הזמנות היום

## מסך 1 — קבלה (טאבלט המפעילה)

### ראש העמוד
- תאריך נוכחי
- סטטיסטיקות: הזמנות פעילות / בהכנה / מוכן / הכנסה כוללת
- פירוט תשלום: מזומן / ביט / פייבוקס / ממתין לתשלום

### טופס הזמנה חדשה (הזמנות דוכן בלבד)
| שדה | פירוט |
|-----|--------|
| שם | טקסט חופשי |
| פריטים | כפתורים ויזואליים עם מחיר — בחירה מרובה |
| כמות | +/- לכל פריט שנבחר |
| סל | סכום כולל מתעדכן אוטומטית |
| הערות | טקסט חופשי |

> **אין שדה "סוג הזמנה"** — כל מה שמוזן ידנית הוא דוכן.
> **אין ממרחים במסך הקבלה** — את מורחת בעל פה בדוכן.
> הזמנות ווצאפ נכנסות אוטומטית ומסומנות אוטומטית כ"ווצאפ".

### רשימת הזמנות — מיון אוטומטי
1. הזמנות מוקדמות דחופות (פחות מ-10 דקות לאיסוף)
2. הזמנות מוקדמות — לפי שעת איסוף
3. הזמנות דוכן — לפי סדר הגעה

### כרטיס הזמנה מציג
- מספר הזמנה (בולט)
- שם + מקור (דוכן / ווצאפ)
- פריטים + כמויות
- ממרחים (הזמנות ווצאפ בלבד)
- סכום לתשלום
- שעת איסוף + טיימר ספירה לאחור
- הערות
- סטטוס תשלום + אמצעי תשלום
- כפתורי סטטוס הכנה

### סטטוסי הכנה
`ממתין` ← `בהכנה` ← `מוכן` ← `נאסף`

- **מוכן** — נשאר גלוי עם מסגרת ירוקה בולטת
- **נאסף** — נעלם לטאב "הושלם"

### התראות חזותיות
- מסגרת אדומה + "דחוף!" — פחות מ-10 דקות לאיסוף
- מסגרת ירוקה — מנה מוכנה לאיסוף

### תשלום על כל הזמנה
בחירה: **מזומן / ביט / פייבוקס / לא שולם**

### עדכון ווצאפ ללקוח
כשהזמנה עוברת ל"מוכן" — כפתור "שלח עדכון ללקוח" שפותח ווצאפ עם הודעה מוכנה (deep link):
> *"היי [שם], הזמנה מספר [X] מוכנה לאיסוף! 🥩"*
את מאשרת בלחיצה אחת.

### תזכורת
10 דקות לפני שעת האיסוף — כפתור "שלח תזכורת" נדלק על הכרטיס

### טאב "הושלם"
ארכיון של כל הזמנות שנאספו באותו יום

### סיכום יום (כפתור)
- כמות מכל פריט
- סה"כ הזמנות ומנות
- סה"כ הכנסה
- פירוט: מזומן / ביט / פייבוקס / ממתין

### היסטוריה
- תצוגת ימים קודמים לפי תאריך
- סיכום כספי לכל יום

## מסך 2 — ציפסרים (טאבלט תצוגה)

### עקרון מרכזי: אין צורך לגעת במסך

- מתרענן אוטומטית דרך WebSocket
- Wake Lock API — מסך לא כובה אוטומטית
- גופן גדול, ניגודיות גבוהה

### מה מוצג
- רק הזמנות בסטטוס **ממתין** ו**בהכנה**
- **שתייה לא מופיעה** — נמסרת מיד בדוכן

### אזור עליון — "להכין עכשיו" (קיבוץ לפי סוג)
```
🍗 שניצל בחלה      ×3
🍗 שניצלונים       ×2
🍟 צ'יפס ספיישל   ×2
🍟 צ'יפס           ×4
```

### אזור תחתון — התראות דחיפות
```
⚠️ הזמנה #3 — דני — 8 דקות לאיסוף
```

### צבעי רקע
- 🟡 רגיל
- 🔴 דחוף (פחות מ-10 דקות)

### מי מעדכן סטטוסים
| סטטוס | מי מעדכן |
|--------|----------|
| ממתין → בהכנה | עובד הקבלה (טאבלט 1) |
| בהכנה → מוכן | ציפסר (טאבלט 2) |
| מוכן → נאסף | עובד הקבלה (טאבלט 1) |

## מסך 3 — טופס לקוח (קישור ווצאפ)

### שעות פעילות
- פעיל רק בימי שלישי 18:00–20:30
- מחוץ לשעות — הודעה: *"הדוכן פתוח בימי שלישי 18:00–20:30"*

### שדות הטופס
1. שם
2. שעת איסוף (בחירה בין 18:00–20:30 בלבד)
3. פריטים — כפתורים ויזואליים עם מחיר ותיאור
4. כמות לכל פריט שנבחר
5. ממרחים לכל פריט שניצל (שתי קבוצות כמפורט)
6. הערות (אופציונלי)
7. כפתור "שליחת הזמנה"

### לאחר שליחה
- הלקוח רואה אישור עם מספר הזמנה + סכום לתשלום
- ההזמנה נכנסת אוטומטית לטאבלט הקבלה בסטטוס "ממתין"
- מסומנת אוטומטית כמקור "ווצאפ"

## הערות לפיתוח

- **WebSocket** — כל שינוי סטטוס מתעדכן בכל המסכים תוך שנייה
- **Offline first** — IndexedDB מקומי + סנכרון לשרת כשחיבור חוזר
- **Wake Lock** — מניעת כיבוי מסך אוטומטי בטאבלט הציפסרים
- **Deep link ווצאפ** — `https://wa.me/[phone]?text=[message]`
- **תשלום** — מעקב בלבד, לא מערכת תשלום דיגיטלית
- **טופס לקוח** — עיצוב נקי, מינימום שדות, קל למילוי בטלפון תוך 30 שניות
- **ארכיטקטורה מוכנה לעתיד** — בנה את ה-API routes בצורה שתאפשר חיבור ווצאפ בשלב 2 בלי לשבור את השלב 1
